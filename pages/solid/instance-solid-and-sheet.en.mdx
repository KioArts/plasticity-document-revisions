# Instance Solid and Sheet

<Reference shortcut="" command="Instance" button="In Command Palette"  buttonIcon="button-instance"/>

Create reusable duplicates of a Solid or Sheet object without copying the underlying geometry. An instance stores transformation data (position, pivot point, rotation, scale), material data and references the original Solid object, so any change to the master geometry is reflected instantly in all its instances. Istances are read-only copies of the parent and cannot be edited independently. Instances aprear with medium to light gray surface edge color in order to differentiate with the parrent object.

- Use the **Instance** command when you need many identical parts such as fasteners, panels, or repetitive structiral elements while keeping the model lightweight and easy to maintain.
- Instances work with array commands such as **Curve Array**, **Radial Array**, and **Rectangular Array**.  
- Instances can also be used with the **Mirror** command.

## Basic Usage

1. Select the Solid or Sheet you want to use as the parent object.
2. Open the Command Palette, type **Instance**, and execute the command.
3. Move the newly created instance to the desired location.
4. Confirm with OK in the Command Dialog or right-click.

## Instance Groups

The process of instancing groups is similar to instancing objects.

When a group is selected and it contains primary geometry, an instance of the group will be created and placed outside of the group folder under its own dedicated category in the Outliner.

### Basic Usage

1. Select a Group in the Outliner to instantiate.
2. Search for *Instance* in the Command Palette and execute it.
3. Move the instanced group.
4. Confirm with OK in the Command Dialog or right-click.

![Instancing a Group Part 1](/img/instance-group-02.png)

| Number | Description |
| :----: | ----------- |
| 1 | Main group with components inside. |
| 2 | Parent objects that will be instanced. |
| 3 | Instance of main group allocated outside of the main group under the category **"Instances"**. |
