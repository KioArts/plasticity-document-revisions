## 説明

ソリッドまたはシートオブジェクトの再利用可能な複製を、基となるジオメトリをコピーすることなく作成します。インスタンスは変換データ（位置、ピボットポイント、回転、スケール）、マテリアルデータを保存し、元のソリッドオブジェクトを参照するため、マスタージオメトリへの変更はすべてのインスタンスに即座に反映されます。

インスタンスは親オブジェクトの読み取り専用のコピーであり、独立して編集することはできません。元のオブジェクトとの区別を明確にするため、中程度から明るいグレーのサーフェスエッジカラーで表示されます。

<Callout type="info">
**注意:**    
1. ファスナー、パネル、繰り返し構造要素など、多数の同一パーツが必要で、かつモデルを軽量かつ管理しやすい状態に保ちたい場合は、**インスタンスを作成**コマンドを使用してください。 
2. インスタンスは、[**カーブ配列**](/common/curve-array)、[**放射状配列**](/common/radial-array)、[**矩形配列**](/common/rectangular-array)などの配列コマンドと連携して使用できます。  
3. インスタンスは[**ミラー**](/common/mirror)コマンドとも使用でき、マスタージオメトリへの変更がすべてのミラーインスタンスに自動的に反映されるライブシンメトリが実現できます。
4. 作成したインスタンスに[**配置**](/common/place)コマンドを使用して、オブジェクト上に配置し、必要に応じてサーフェスの詳細を手動で追加できます。
5. 現時点では、作成したインスタンスに[**配置してコピー**](/common/copy-with-placement)コマンドを使用することはできません。
</Callout>

## 基本的な使い方

1. 親オブジェクトとして使用するソリッドまたはシートを選択します。
2. コマンドパレットを開き、**インスタンスを作成**と入力してコマンドを実行します。
3. 新しく作成されたインスタンスを目的の場所に移動します。
4. コマンドダイアログでOKをクリックするか、右クリックして確定します。

<Callout type="info">
**注意:** インスタンスが作成されたら、そのインスタンスを何度でも複製することができます。すべての複製は、元の親オブジェクトへの変更を引き続き反映します。    
</Callout>

## インスタンスグループ

グループのインスタンス化のプロセスは、オブジェクトのインスタンス化と同様です。

グループが選択されており、そのグループにプライマリジオメトリが含まれている場合、グループのインスタンスが作成され、グループフォルダの外側に、アウトライナー内の専用カテゴリの下に配置されます。

<Center width="600px">
![グループのインスタンス化 パート1](/img/instance-group-solid.png)
</Center>

| 番号 | 説明 |
| :----: | ----------- |
| 1 | コンポーネントを含むメイングループ。 |
| 2 | インスタンス化される親オブジェクト。 |
| 3 | **「インスタンス」**カテゴリのもとメイングループの外側に配置された、メイングループのインスタンス。 |

### 基本的な使い方

1. アウトライナーでインスタンス化するグループを選択します。
2. コマンドパレットで**インスタンスを作成**を検索し、コマンドを実行します。
3. インスタンス化されたグループを移動します。
4. コマンドダイアログでOKをクリックするか、右クリックして確定します。
